# Commit Changes to GitHub

Commit the current changes to the repository.

## Instructions

1. Run `git status` to see what files have changed
2. Run `git diff` to review the actual changes
3. Run `git log --oneline -5` to see recent commit message style
4. Stage all relevant changes with `git add`
5. Create a commit with a clear, concise message summarizing the changes
6. If an argument is provided, use it as the commit message: $ARGUMENTS

Keep commit messages short (1-2 sentences) and focused on what changed and why.
